## Design Philosophy

### Every Shader Should Serve All Three Pillars

All work in this repository should meaningfully engage with all three dimensions:

1. **Start with wonder**: What psychedelic element will make this captivating?
2. **Refine with elegance**: How can we make this genuinely beautiful?
3. **Distinguish with strangeness**: What unexpected twist makes this memorable and thought-provoking?

The best shaders achieve a balance where all three elements amplify each other.

### Technical Excellence Serves Artistic Vision

- Code is clean, well-documented, and maintainable
- Performance is optimized to allow complex visual ideas to render smoothly
- Algorithms are chosen for both their visual output and their conceptual resonance
- Parameters are intuitive and allow for compelling customization

### Iteration and Refinement

- Test shaders in various contexts and lighting conditions
- Gather feedback on emotional resonance, not just technical execution
- Iterate toward coherence‚Äîwhere technical implementation feels like the inevitable vehicle for the artistic idea

## Thematic Categories

### Color & Chromatic
Shaders exploring extreme color relationships, iridescence, impossible color combinations, and perceptual color theory.

**Example goals**: Synesthetic experiences, color fields that seem to vibrate or shift, gradients that violate normal color theory.

### Form & Geometry
Shaders that explore non-Euclidean space, fractals, tessellations, and geometric transformation.

**Example goals**: Impossible objects, recursive beauty, systems that suggest intelligent design, forms that morph hypnotically.

### Liquid & Fluid
Shaders simulating or suggesting fluid dynamics, flows, and organic motion.

**Example goals**: Otherworldly substances, impossible physics, movements that feel alive, surfaces that ripple with intention.

### Particle & Organism
Shaders treating visual elements as responsive systems‚Äîschools, swarms, cellular automata, and living-seeming behaviors.

**Example goals**: Emergent complexity, intelligent-seeming patterns, systems with personality, visual growth and decay.

### Light & Luminosity
Shaders focused on light propagation, glow, shadow, and illumination effects.

**Example goals**: Transcendent luminescence, impossible light sources, atmospheres that feel supernatural, volume and depth.

### Noise & Randomness
Shaders that use mathematical noise functions and randomization as primary generators.

**Example goals**: Organic-seeming variation, worlds of detail, fractured beauty, structured chaos.

## Technical Principles

### 1. Mathematical Elegance
- Prefer algorithms with simple, beautiful underlying mathematics
- Seek solutions where the visual output naturally follows from the conceptual framework
- Use mathematical constants and ratios that create inherent harmony

### 2. Perceptual Optimization
- Consider how the human visual system processes information
- Use color relationships that feel harmonious even when saturated
- Create focal points that guide attention intentionally

### 3. Parametric Expressiveness
- Design shaders with control parameters that feel intuitive
- Allow for meaningful variation‚Äîparameters should create perceptually distinct results
- Document what each parameter does in poetic and technical terms

### 4. Interdimensional Coherence
- Ensure that different shaders in the same category share a visual language
- Allow shaders to be composed and layered harmoniously
- Create a world that feels internally consistent despite its strangeness

## Quality Standards

### Visual Quality Checklist
- [ ] Does this create a moment of genuine awe or delight?
- [ ] Would I return to this multiple times to experience it again?
- [ ] Does it feel both deliberate and inevitable?
- [ ] Is there beauty in the mathematics underneath?
- [ ] Does it surprise me, even after multiple viewings?

### Code Quality Checklist
- [ ] Is the code well-commented and conceptually clear?
- [ ] Are variable names meaningful and poetic where appropriate?
- [ ] Has the shader been tested on target hardware?
- [ ] Are there obvious areas for optimization without sacrificing visual quality?
- [ ] Would another developer understand the artistic intent from the code?
