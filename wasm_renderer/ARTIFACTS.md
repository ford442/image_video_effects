# WASM Artifacts — Locations & Strategy

## Artifact strategy (Phase 2)

We use **CI-built, committed baseline**:

| Artifact | Location | Role |
|----------|----------|------|
| C++ sources | `wasm_renderer/*.cpp`, `*.h` | Source of truth for renderer logic |
| JS bridge | `wasm_renderer/wasm_bridge.js` | **Canonical bridge — edit here only** |
| Bridge types | `wasm_renderer/wasm_bridge.d.ts` | Canonical TypeScript declarations |
| Emscripten output | `public/wasm/pixelocity_wasm.{js,wasm}` | Runtime-loaded WASM module |
| Bridge copies | `src/wasm/wasm_bridge.js`, `public/wasm/wasm_bridge.js` | Generated by `build.sh` — do not edit |

**Why committed binaries?** So `npm start` works on machines without Emscripten. CI always rebuilds from C++ and fails if sources change without a matching build.

**Not using Git LFS** — artifacts are ~200 KB total; CI rebuild + validation is sufficient drift protection.

## Canonical vs generated paths

```
wasm_renderer/wasm_bridge.js     ← EDIT HERE
        │
        ├─ build.sh copies ──► src/wasm/wasm_bridge.js      (WASMRenderer.ts webpack import)
        └─ build.sh copies ──► public/wasm/wasm_bridge.js   (useWASM runtime dynamic import)

wasm_renderer/main.cpp + renderer.cpp
        │
        └─ emcc ──► public/wasm/pixelocity_wasm.{js,wasm}
```

## Build commands

```bash
npm run wasm:build      # Requires emsdk; fails if emcc missing
npm run wasm:validate   # Checks sizes, magic, exports, bridge sync
npm run wasm:clean      # Remove build/ and public/wasm/
```

### Skipping WASM build intentionally

Headless dev VMs without Emscripten can use committed artifacts:

```bash
SKIP_WASM_BUILD=1 npm run build
```

CI jobs that consume pre-built artifacts from the `wasm` job set this automatically.

## CI pipeline

1. **`wasm` job** — installs emsdk (`mymindstorm/setup-emsdk@v14`), runs `wasm:build`, `wasm:validate`, WASM Jest smoke tests, freshness check, uploads artifacts.
2. **`test` job** — downloads WASM artifacts, runs full unit tests + CRA build with `SKIP_WASM_BUILD=1`, re-validates artifacts.
3. **`test-wasm-e2e` job** — downloads artifacts, production build, Playwright smoke tests (`tests/wasm-renderer.smoke.spec.ts`).

See [`WASM_BUILD_CI_GUIDE.md`](../WASM_BUILD_CI_GUIDE.md) for troubleshooting.
