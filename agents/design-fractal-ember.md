# Fractal Ember 设计文档（参考标准）

> 用于验收 Kimi Claw 生成的 Fractal Ember 着色器

## 基本信息
- **着色器名称**：Fractal Ember
- **视觉家族**：分形晶格 / 几何结构
- **目标用途**：Generative Showcase 核心展示着色器（与 Ethereal Silk 形成视觉对比）

## 1. 整体视觉概念
- 发光的尖锐几何晶格结构，像燃烧的晶体或熔化的几何框架。
- 结构带有分形递归特征，从中心向外延伸，每层细节逐渐变精细。
- 色调以炽热的橙、金、深红、熔岩色为主，带有明显的 "燃烧感" 和 "能量感"。
- 对比 Ethereal Silk 的柔和流动，Fractal Ember 应该是 **锐利、精确、有力量感** 的。

## 2. Idle 状态（未锁定）核心要求
- **运动**：缓慢旋转 + 轻微脉动。几何框架像呼吸一样轻微缩放和亮度变化。
- **视觉吸引力**：12 秒内不能让人觉得无聊。分形结构应该有持续但不突兀的自我演化感。
- **复杂度**：中高复杂度，强调几何秩序和分形美感，而不是混乱的噪声。
- **Hypnotic 感**：需要让人愿意一直看下去，像在看缓慢燃烧的晶体或永恒旋转的几何曼陀罗。

## 3. Mouse Claim 状态（锁定后）核心要求
- 鼠标点击位置产生 **"击碎"** 效果——几何晶格在点击处碎裂成碎片。
- 碎片向外炸开，带有火焰/余烬拖尾，然后缓慢重新聚合重组。
- 重组过程中，几何结构应该有 "愈合" 感，像熔化的金属重新凝固。
- Claim 后的变化要 **明显且令人满足**，但不要破坏整体的几何秩序感。
- 鼠标移动时，附近的晶格结构应该被 "吸引" 或 "扭曲"，产生力场效果。

## 4. 音频反应映射建议

| 音频频段 | 映射目标 | 建议强度 | 说明 |
|----------|-------------------|----------|--------------------------|
| **Bass** | 碎裂/爆炸强度 + 震动感 | 高 | 低频触发碎裂和余烬飞溅 |
| **Mid** | 分形旋转速度 + 脉动频率 | 中 | 中频驱动整体旋转和呼吸 |
| **Treble** | 边缘闪烁 + 高光细节 | 中高 | 高频增加晶格边缘的火花和闪烁 |

## 5. 参数建议（zoomParam1-4）
- `zoomParam1`：分形旋转速度 + 整体脉动幅度
- `zoomParam2`：碎裂强度 + 余烬拖尾长度
- `zoomParam3`：分形递归深度 / 细节层级
- `zoomParam4`：边缘高光 + 火花密度

## 6. 技术与展示要求
- 必须符合 `WGSL_BUILTINS_GENERATIVE.md`
- 分形计算要性能友好，避免过度递归（建议最大 4-5 层迭代）
- Idle 时性能要好，适合长时间轮播
- Claim 时碎裂效果可以短暂增加计算量，但重组后要迅速恢复
- 颜色要炽热、高级，不能太"卡通"或廉价
- 几何结构在碎裂后重组时要有 "记忆"——不要完全随机重组，而是大致恢复原结构

## 7. 验收标准（对照 checklist）
- [ ] Idle 12 秒内持续吸引人，几何结构有自演化感
- [ ] 运动自然流畅，旋转 + 脉动有呼吸感
- [ ] 颜色炽热高级（橙/金/深红/熔岩色）
- [ ] Claim 时碎裂/爆炸效果明显且令人满足
- [ ] 碎片有拖尾/余烬效果，重组过程自然
- [ ] Bass 控制碎裂强度，Mid 控制旋转，Treble 控制边缘火花
- [ ] naga 验证通过，无编译错误
- [ ] JSON 配置完整，含 showcase 标签
- [ ] 性能良好，60fps 稳定（分形计算不过度）
