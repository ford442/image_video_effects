## Immutable Rules (ALL agents)

1. **DO NOT** modify `Renderer.ts`, `types.ts`, bind groups, or install npm packages.
2. **DO NOT** change `@workgroup_size` on shaders using `var<workgroup>`, `@builtin(local_invocation_id)` size math, or `workgroupBarrier()`.
3. **COPY EXACTLY** the 13-binding header and `Uniforms` struct (see below).
4. **PRESERVE** the shader's algorithmic core — upgrades are additive, not rewrites.
5. Update both WGSL header `Features:` and JSON `features` array when adding capabilities.
6. Set `Upgraded: 2026-06-06` (or today's date) in the WGSL header.
7. Run `node scripts/generate_shader_lists.js` after JSON changes.

### 13-Binding Contract (immutable)

```wgsl
@group(0) @binding(0) var u_sampler: sampler;
@group(0) @binding(1) var readTexture: texture_2d<f32>;
@group(0) @binding(2) var writeTexture: texture_storage_2d<rgba32float, write>;
@group(0) @binding(3) var<uniform> u: Uniforms;
@group(0) @binding(4) var readDepthTexture: texture_2d<f32>;
@group(0) @binding(5) var non_filtering_sampler: sampler;
@group(0) @binding(6) var writeDepthTexture: texture_storage_2d<r32float, write>;
@group(0) @binding(7) var dataTextureA: texture_storage_2d<rgba32float, write>;
@group(0) @binding(8) var dataTextureB: texture_storage_2d<rgba32float, write>;
@group(0) @binding(9) var dataTextureC: texture_2d<f32>;
@group(0) @binding(10) var<storage, read_write> extraBuffer: array<f32>;
@group(0) @binding(11) var comparison_sampler: sampler_comparison;
@group(0) @binding(12) var<storage, read> plasmaBuffer: array<vec4<f32>>;

struct Uniforms {
  config: vec4<f32>,       // x=Time, y=MouseClickCount, z=ResX, w=ResY
  zoom_config: vec4<f32>,  // x=Time, y=MouseX, z=MouseY, w=MouseDown
  zoom_params: vec4<f32>,  // x=Param1, y=Param2, z=Param3, w=Param4
  ripples: array<vec4<f32>, 50>,
};
```

## Reusable Code Chunks

### Chunk: acesToneMap

Insert once before `@compute @workgroup_size`. Do not duplicate if `aces_tonemap` or `acesToneMap` already exists.

```wgsl
// ═══ CHUNK: acesToneMap (standard upgraded-rgba) ═══
fn acesToneMap(x: vec3<f32>) -> vec3<f32> {
  return clamp((x * (2.51 * x + 0.03)) / (x * (2.43 * x + 0.59) + 0.14), vec3<f32>(0.0), vec3<f32>(1.0));
}
```

### Chunk: audio reads

```wgsl
let bass   = plasmaBuffer[0].x;
let mids   = plasmaBuffer[0].y;
let treble = plasmaBuffer[0].z;
```

### Chunk: temporal feedback

Place **after** color computation, **before** chromatic/ACES:

```wgsl
// ═══ CHUNK: temporal-feedback (dataTextureC → dataTextureA) ═══
let prev = textureSampleLevel(dataTextureC, u_sampler, uv, 0.0);
col = mix(col, prev.rgb * 0.92, 0.05 + bass * 0.01);
```

Adjust blend: `0.05` = subtle trails, `0.15` = heavy persistence. Scale with `bass` for audio-reactive smear.

### Chunk: chromatic aberration

Place **after** temporal, **before** ACES:

```wgsl
// ═══ CHUNK: chromatic-aberration ═══
let caStr = 0.003 * (1.0 + bass) + effectStrength * 0.001;
col = vec3<f32>(col.r + caStr, col.g, col.b - caStr * 0.5);
```

Replace `effectStrength` with the shader's natural intensity variable (SDF shell, fog density, glow, thickness, etc.).

### Chunk: depth-aware effect modulation

Use when shader runs in chained slots and should respect foreground:

```wgsl
// ═══ CHUNK: depth-aware-compositing ═══
let depth = textureSampleLevel(readDepthTexture, non_filtering_sampler, uv, 0.0).r;
let depthMod = 1.0 - exp(-depth * 2.0);  // 0=far, 1=near
col = mix(col, col * (0.7 + depthMod * 0.3), 0.5);
```

### Chunk: mouse attractor (generative)

Many generative shaders ignore mouse. Add subtle attraction:

```wgsl
// ═══ CHUNK: mouse-attractor ═══
let mouse = (u.zoom_config.yz - 0.5) * vec2<f32>(res.x / res.y, 1.0);
let md = length(uv - mouse);
let attract = exp(-md * 2.0) * 0.08;
// Apply to position: cx += (mouse.x - cx) * attract;
```

Reference: `gen-protocell-division.wgsl` lines 101–108.

### Chunk: semantic alpha + final output

```wgsl
// ═══ CHUNK: upgraded-rgba-output ═══
col = acesToneMap(col * 1.1);  // use 1.5 for high-dynamic generative
let alpha = clamp(effectStrength * 2.0 + coreGlow * 0.5, 0.0, 1.0);
textureStore(writeTexture, coord, vec4<f32>(col * alpha, alpha));
textureStore(writeDepthTexture, coord, vec4<f32>(effectStrength * 0.5, 0.0, 0.0, 0.0));
textureStore(dataTextureA, coord, vec4<f32>(col, alpha));
```

**Critical:** `dataTextureA` must store **pre-multiplied or straight RGB matching what temporal will read next frame**. Batch 2 uses straight `col` (not `col * alpha`) in `dataTextureA` — match the reference shader's convention.

### Chunk: bass envelope (anti-strobe)

For audio-reactive shaders that flicker:

```wgsl
fn bassEnv(prev: f32, bass: f32) -> f32 {
  let k = select(0.15, 0.8, bass > prev);  // attack / release
  return mix(prev, bass, k);
}
// Read prev bass from dataTextureC.r, write new envelope to dataTextureA.r
```

## Batch 3A — Chromatic-Only (5 shaders)

**Role:** Post-FX Specialist  
**Target:** Shaders with ACES + dataA + temporal + audio + depth, but `has_chromatic: false` in audit.

| # | Shader ID | File |
|---|-----------|------|
| 1 | `gen-translucent-nebula` | `public/shaders/gen-translucent-nebula.wgsl` |
| 2 | `gen-alpha-aurora` | `public/shaders/gen-alpha-aurora.wgsl` |
| 3 | `gen-ghost-flame` | `public/shaders/gen-ghost-flame.wgsl` |
| 4 | `gen-prismatic-crystal-growth` | `public/shaders/gen-prismatic-crystal-growth.wgsl` |
| 5 | `electric-eel-storm` | `public/shaders/electric-eel-storm.wgsl` |

### Task per shader

1. Find the color variable immediately before `textureStore(writeTexture, ...)`.
2. Insert temporal feedback chunk (if `dataTextureC` read missing).
3. Insert chromatic aberration chunk using the shader's glow/density variable.
4. Ensure `acesToneMap` wraps final color.
5. Add `chromatic-aberration` to header `Features:`.
6. No JSON changes unless `features` array is out of sync.

### Example insertion point

```wgsl
// BEFORE (typical):
var col = nebulaColor(uv, t, bass);
col = acesToneMap(col * 1.1);
textureStore(writeTexture, coord, vec4<f32>(col, alpha));

// AFTER:
var col = nebulaColor(uv, t, bass);
let prev = textureSampleLevel(dataTextureC, u_sampler, uv, 0.0);
col = mix(col, prev.rgb * 0.92, 0.05 + bass * 0.01);
let caStr = 0.003 * (1.0 + bass) + glow * 0.001;
col = vec3<f32>(col.r + caStr, col.g, col.b - caStr * 0.5);
col = acesToneMap(col * 1.1);
textureStore(writeTexture, coord, vec4<f32>(col, alpha));
textureStore(dataTextureA, coord, vec4<f32>(col, alpha));
```

## Batch 3B — dataTextureA Plumber (10 shaders)

**Role:** Temporal Plumber  
**Target:** Generative shaders that read `dataTextureC` but **do not** write `dataTextureA`.

Run to enumerate targets:

```bash
node -e "
const fs=require('fs'),path=require('path');
for(const f of fs.readdirSync('shader_definitions/generative').filter(x=>x.endsWith('.json'))){
  const j=JSON.parse(fs.readFileSync('shader_definitions/generative/'+f));
  const w=fs.readFileSync(path.join('public',j.url),'utf8');
  if(w.includes('dataTextureC') && !w.includes('textureStore(dataTextureA'))
    console.log(j.id);
}
"
```

### Task per shader

1. Add `textureStore(dataTextureA, coord, vec4<f32>(col, alpha))` after `writeTexture` store.
2. If temporal read is missing, add temporal feedback chunk.
3. Add `temporal-feedback` to header and JSON `features`.
4. Verify `writeDepthTexture` store still present.

### Notes file

Write `agents/swarm-outputs/kimi-notes/{shader-id}.notes.kimi.md` per shader.

## Batch 3C — ACES Gap Fill (10 shaders)

**Role:** Tone Mapper  
**Target:** Generative shaders where JSON has `upgraded-rgba` but WGSL lacks `acesToneMap`.

```bash
node -e "
const fs=require('fs'),path=require('path');
for(const f of fs.readdirSync('shader_definitions/generative').filter(x=>x.endsWith('.json'))){
  const j=JSON.parse(fs.readFileSync('shader_definitions/generative/'+f));
  if(!(j.features||[]).includes('upgraded-rgba')) continue;
  const w=fs.readFileSync(path.join('public',j.url),'utf8');
  if(!/fn acesToneMap|aces_tonemap/i.test(w)) console.log(j.id);
}
"
```

### Task per shader

1. Add `acesToneMap` function chunk.
2. Wrap color in `acesToneMap((color) * 1.1)` at final output.
3. Add `aces-tone-map` to header `Features:`.
4. Do not duplicate if `aces_tonemap` exists — rename to `acesToneMap` for consistency.

**Role:** Simulation Architect  
**Goal:** Algorithmic depth, not just post-FX stacking.

| # | Shader ID | Upgrade direction |
|---|-----------|-------------------|
| 1 | `gen_reaction_diffusion` | Pass 1: simulate RG channels in dataA; Pass 2: colorize (pair with existing or inline two-pass via dataB) |
| 2 | `gen-murmuration-phantom` | Move boid positions to `extraBuffer`; render trails via dataA |
| 3 | `gen-navier-stokes-ink` | Velocity field in dataB, pressure solve stub, advect via dataC |

| 5 | `gen-conway-game-of-life` | CA state in dataA.r, generation count in dataA.g, fade trails in dataC |

### Multi-pass pattern (single shader, two data textures)

```wgsl
// Pass 1 state (written this frame):
textureStore(dataTextureA, coord, vec4<f32>(newState, generation, 0.0, 1.0));

// Pass 2 read previous state:
let state = textureSampleLevel(dataTextureC, u_sampler, uv, 0.0);
let alive = state.r > 0.5;
var col = select(deadColor, aliveColor, alive);
```

Mark JSON with `"features": ["multi-pass-1", "temporal-feedback"]` if creating a shader pair.

## Validation Checklist (orchestrator runs after each batch)

```bash
# Project-level
node scripts/generate_shader_lists.js
node scripts/check_duplicates.js

# Per-shader (if naga CLI available)
naga public/shaders/{shader-id}.wgsl

# Feature audit
node -e "
const fs=require('fs'),path=require('path');
const id=process.argv[1];
const j=JSON.parse(fs.readFileSync('shader_definitions/generative/'+id+'.json'));
const w=fs.readFileSync(path.join('public',j.url),'utf8');
const checks={
  aces:/fn acesToneMap|aces_tonemap/i.test(w),
  dataA:w.includes('textureStore(dataTextureA'),
  chromatic:/caStr|chromatic/i.test(w),
  temporal:w.includes('dataTextureC'),
  header:w.includes('upgraded-rgba'),
};
console.log(id, checks, Object.values(checks).every(Boolean)?'PASS':'FAIL');
" gen-translucent-nebula
```
