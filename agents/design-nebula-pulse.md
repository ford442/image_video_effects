# Nebula Pulse 设计文档（参考标准）

> 用于验收 Kimi Claw 生成的 Nebula Pulse 着色器

## 基本信息
- **着色器名称**：Nebula Pulse
- **视觉家族**：粒子能量场 / 星云
- **目标用途**：Generative Showcase 核心展示着色器（粒子系统代表作）

## 1. 整体视觉概念
- 由无数微小粒子组成的能量云/星云，像深空中的星爆或能量场。
- 粒子不是随机乱飞，而是有 **组织性**——形成旋涡、波纹、或脉动的球状结构。
- 色调以深紫、靛蓝、电蓝、亮白为主，带有明显的 "宇宙感" 和 "能量感"。
- 对比 Ethereal Silk 的柔和和 Fractal Ember 的锐利，Nebula Pulse 应该是 **有机、弥漫、充满能量** 的。

## 2. Idle 状态（未锁定）核心要求
- **运动**：缓慢呼吸式的粒子扩散与收缩。整体像一颗巨大的能量心脏在跳动。
- **视觉吸引力**：12 秒内不能让人觉得无聊。粒子应该形成不断演化的图案——有时是旋涡，有时是波纹，有时是向外扩散的球面波。
- **复杂度**：中复杂度，粒子数量要足够多（视觉上密集），但计算要保持性能友好。
- **Hypnotic 感**：需要让人愿意一直看下去，像在看深空中的星云呼吸或能量场脉动。

## 3. Mouse Claim 状态（锁定后）核心要求
- 鼠标附近粒子被 **吸引聚集**，形成更亮、更密集的核心。
- 聚集过程中，粒子应该有 "兴奋" 感——亮度增加、运动速度加快、轨迹拖尾变长。
- 鼠标移动时，聚集的核心跟随移动，像一颗被拖拽的微型恒星。
- 松开鼠标后，粒子缓慢扩散回原始状态，不要瞬间消失。
- Claim 后的变化要 **明显且令人满足**，但不要破坏整体的星云美感。

## 4. 音频反应映射建议

| 音频频段 | 映射目标 | 建议强度 | 说明 |
|----------|-------------------|----------|--------------------------|
| **Bass** | 粒子聚集强度 + 脉动幅度 | 高 | 低频让星云"心跳"更强烈 |
| **Mid** | 粒子运动速度 + 旋涡频率 | 中 | 中频驱动整体流动和旋涡 |
| **Treble** | 粒子闪烁 + 能量爆发 | 中高 | 高频触发粒子亮度和能量爆发 |

## 5. 参数建议（zoomParam1-4）
- `zoomParam1`：粒子扩散/收缩速度 + 整体脉动频率
- `zoomParam2`：粒子聚集密度 + 核心亮度
- `zoomParam3`：旋涡复杂度 / 图案演化速度
- `zoomParam4`：粒子拖尾长度 + 能量爆发强度

## 6. 技术与展示要求
- 必须符合 `WGSL_BUILTINS_GENERATIVE.md`
- 粒子系统要性能友好，避免 O(n²) 计算（建议用噪声场驱动粒子运动，而不是粒子间交互）
- Idle 时性能要好，适合长时间轮播
- Claim 时聚集效果可以短暂增加计算量，但松开后要迅速恢复
- 颜色要深邃、高级，不能太"塑料"或廉价
- 粒子运动要有 "记忆"——不要完全随机，而是受噪声场引导，形成可预测的美感

## 7. 验收标准（对照 checklist）
- [ ] Idle 12 秒内持续吸引人，粒子形成不断演化的图案
- [ ] 运动自然流畅，有呼吸/脉动感
- [ ] 颜色深邃高级（深紫/靛蓝/电蓝/亮白）
- [ ] Claim 时粒子聚集效果明显且令人满足
- [ ] 聚集核心有亮度/速度增强，松开后自然扩散
- [ ] Bass 控制聚集强度，Mid 控制旋涡，Treble 控制闪烁爆发
- [ ] naga 验证通过，无编译错误
- [ ] JSON 配置完整，含 showcase 标签
- [ ] 性能良好，60fps 稳定（粒子数量不过度）
