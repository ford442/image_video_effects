## Upgrade Protocol (Grok Style)

For each shader in the batch, do the following:

### 1. Creative Diagnosis (write this down first)
- What is the current shader's strongest emotional or visual quality?
- What is it *almost* doing that would be transcendent if pushed 20% further?
- What single surprising behavior (mouse + audio + depth interaction) would make someone say "I've never seen anything like that in real time before"?

### 2. High-Signal Upgrade Direction
Choose **one** primary creative lever per shader (do not try to do everything):
- A new interaction metaphor (mouse as "keystone species", "sediment disturbance", "external magnetic field")
- A new temporal or state behavior (slow time-dilated growth, seasonal cycles via plasmaBuffer, extinction events)
- A new visual language (glyph formation, membrane tearing with color transmission, readable text from fluid)
- A new compositing relationship (how this shader wants to live in slot 1 vs slot 2/3)

### 3. Execution
- You may write full new WGSL sections or large refactored blocks.
- You **must** respect the 13-binding contract and hot-swap rules.
- You should include the full Standard Hybrid Header with honest "Chunks From" attribution.
- You are allowed (and encouraged) to introduce new helper functions if they serve the creative vision — just keep them well-commented.
- Audio reactivity via `plasmaBuffer` and depth via `readDepthTexture` should be first-class citizens in every upgrade.

### 4. Deliverables
For each shader produce:
- The upgraded `.wgsl` (or major sections if doing staged work)
- A rich `swarm-outputs/grok-upgrades/<shader-id>.grok-upgrade.md` containing:
  - Original creative diagnosis (2–3 sentences)
  - The upgrade direction you chose and why
  - The key new behaviors introduced
  - "This is the moment it sings" — the exact mouse + audio + time combination that reveals the new soul
  - Suggested JSON updates (params, features, tags)
  - One paragraph for future Claude: "This shader is now conceptually complete — it will benefit from X optimization pass later"
